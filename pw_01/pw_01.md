## Блок А. Анализ времени и Дат
### Задание 1. Дни недели продаж. Определите, в какой день недели (понедельник, вторник и т.д.) совершается наибольшее количество продаж (sales). Выведите день недели и количество транзакций.

Скрипт запроса:
```sql
SELECT 
    CASE EXTRACT(DOW FROM sales_transaction_date)
        WHEN 0 THEN 'Воскресенье'
        WHEN 1 THEN 'Понедельник'
        WHEN 2 THEN 'Вторник'
        WHEN 3 THEN 'Среда'
        WHEN 4 THEN 'Четверг'
        WHEN 5 THEN 'Пятница'
        WHEN 6 THEN 'Суббота'
    END AS day_of_week,
    COUNT(*) AS number_of_sales
FROM sales
GROUP BY EXTRACT(DOW FROM sales_transaction_date)
ORDER BY number_of_sales DESC
LIMIT 1;
```

Пояснение:
- EXTRACT(DOW FROM ...) – извлекает номер дня недели (0 = воскресенье, 6 = суббота)
- CASE – преобразует номер в читаемое название
- GROUP BY – группирует по дню недели
- ORDER BY ... DESC LIMIT 1 – показывает день с максимумом продаж

## Блок Б. Геопространственный анализ
### Задание 8. Покрытие дилеров. Найдите дилерский центр, у которого наибольшее количество клиентов в радиусе 100 миль.

Скрипт запроса:
```sql
SELECT 
    d.dealership_id,
    d.city,
    d.state,
    COUNT(DISTINCT c.customer_id) AS customers_in_radius
FROM dealerships d
CROSS JOIN customers c
WHERE 
    c.latitude IS NOT NULL 
    AND c.longitude IS NOT NULL
    AND d.latitude IS NOT NULL 
    AND d.longitude IS NOT NULL
    AND point(c.longitude, c.latitude) <@> point(d.longitude, d.latitude) <= 100  -- расстояние в милях
GROUP BY d.dealership_id, d.city, d.state
ORDER BY customers_in_radius DESC
LIMIT 1;
```

Пояснение:
- CROSS JOIN – соединяет каждого клиента с каждым дилером
- point(long, lat) <@> point(long, lat) – вычисляет расстояние в милях между двумя точками на Земле
- <= 100 – фильтр: клиенты в радиусе 100 миль
- COUNT(DISTINCT c.customer_id) – считаем уникальных клиентов для каждого дилера
- ORDER BY ... DESC LIMIT 1 – выбираем дилера с максимальным покрытием

## Блок В. Сложные типы (Массивы и JSON)
### Задание 11. История покупок в JSON. Создайте представление или запрос, который формирует JSON-объект для каждого клиента: { "id": 1, "name": "Ivan", "products": ["Car", "Scooter"] }, используя агрегацию массивов.

Скрипт запроса:
```sql
SELECT 
    jsonb_build_object(
        'id', c.customer_id,
        'name', c.first_name || ' ' || c.last_name,
        'products', COALESCE(
            (SELECT jsonb_agg(DISTINCT p.product_type)
             FROM sales s
             JOIN products p ON s.product_id = p.product_id
             WHERE s.customer_id = c.customer_id),
            '[]'::jsonb
        )
    ) AS customer_history
FROM customers c
WHERE EXISTS (
    SELECT 1 FROM sales s2 WHERE s2.customer_id = c.customer_id
)
LIMIT 5;
```

Пояснение:
- jsonb_build_object() – собирает JSON-объект из пар ключ-значение
- jsonb_agg(DISTINCT ...) – агрегирует уникальные продукты в массив JSON
- COALESCE(..., '[]'::jsonb) – если продуктов нет, возвращает пустой массив
- EXISTS – фильтрует клиентов без покупок

## Блок Г. Текстовая аналитика
### Задание 16. Частотный словарь. Составьте топ-10 самых часто встречающихся слов в таблице customer_survey (столбец feedback), исключив слова короче 3 символов.

Скрипт запроса:
```sql
SELECT 
    word,
    COUNT(*) AS frequency
FROM (
    SELECT 
        UNNEST(
            STRING_TO_ARRAY(
                REGEXP_REPLACE(
                    LOWER(feedback), 
                    '[^a-zа-яё\s]', 
                    '', 
                    'g'
                ), 
                ' '
            )
        ) AS word
    FROM customer_survey
    WHERE feedback IS NOT NULL AND feedback != ''
) AS words
WHERE 
    LENGTH(word) >= 3
    AND word != ''
GROUP BY word
ORDER BY frequency DESC
LIMIT 10;
```

Пояснение:
- LOWER() – приводит текст к нижнему регистру
- REGEXP_REPLACE(..., '[^a-zа-яё\s]', '', 'g') – удаляет все символы, кроме букв и пробелов (поддержка английского и русского)
- STRING_TO_ARRAY(..., ' ') – разбивает текст на массив слов по пробелам
- UNNEST() – разворачивает массив в строки (одно слово на строку)
- LENGTH(word) >= 3 – исключает короткие слова (предлоги, союзы)
- GROUP BY ... ORDER BY ... DESC LIMIT 10 – топ-10 самых частотных слов
